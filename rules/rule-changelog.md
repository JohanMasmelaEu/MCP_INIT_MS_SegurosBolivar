# Changelog — Registro Automatico de Cambios

## Estrategia por Rama
Cada rama genera su archivo en `.kiro/changelogs/changelog-<nombre-rama>.md`.

## Categorias (en espanol)
| Categoria | Uso |
|-----------|-----|
| Agregado | Funcionalidades nuevas |
| Cambiado | Cambios en funcionalidades existentes |
| Obsoleto | Funcionalidades que se eliminaran pronto |
| Eliminado | Funcionalidades eliminadas |
| Corregido | Correccion de bugs |
| Seguridad | Correccion de vulnerabilidades |

## Reglas
- Iniciar con `- `. Tiempo pasado: "Se agrego", "Se corrigio"
- Referenciar tickets Jira cuando aplique: (JIRA-1234)
- Nunca borrar archivos de changelog existentes
- Nunca sobrescribir entradas existentes
